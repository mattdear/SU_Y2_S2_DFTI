Page 1

Structure of an HTML Web Page

  • We start with a DOCTYPE declaring the version of HTML you are using, e.g: 
    <!DOCTYPE html>
    This declares that we are using the latest version of HTML (currently HTML5)
  • Other DOCTYPEs exist, e.g. for XHTML
  • The actual page always begins with the tag <html>
  • Consists of head and body sections, defined by the <head> and <body> tags
  • The head section contains the page title and other supplementary information
  • The body section contains the actual content
  • We always need to close sections, e.g. </head>, </body> and </html>

Page 2

A very simple HTML page

  • e.g: 
    <!DOCTYPE html>
<html>
<head>
<title>My Home Page</title>
</head>
<body>
Welcome to my  home page!!!
</body>
</html>
  • This gives us the result: 
    Welcome to my  home page!!!

Page 3

Page elements

  • A page element is any item on the page, such as a paragraph, heading or image
  • Consists of opening and closing tags to start and finish a page element
  • For example, for a paragraph, we use <p< and </p>
  • e.g. 
    <p>
Search for your favourite top 40 singles and albums with HitTastic! 
Whether it's pop, rock or rap you're into, no-one has a wider selection 
of hits than HitTastic!
</p>

Page 4

Common tags

  • <p> : paragraph of text
  • <h1 -h6> : headings (h1 largest, h6 smallest)
  • <header> : A page area for a page's header (containing the title, etc)
  • <nav> : A navigation bar
  • <section> : A generic page section. Formerly <div> was used for this purpose
  • <footer> : A page footer

Page 5

Well-written HTML pages

Well-written, modern HTML pages should follow these recommendations

  • Write all tags in lower case: <body> not <BODY>
  • Close all tags: no <p> without </p> !
    • If it does not make sense to have a closing tag, e.g. <br>, open and close the tag in one go e.g. <br />

Lists

  • Unordered (<ul>) lists for bullet-points
  • Ordered(<ol>) lists for numbered points
  • <li> (list item) for each individual item in the list
  • For example:

<ul>
<li>Firefox </li>
<li>Chrome </li>
<li>Safari </li>
<li>IE </li>
<li>Opera</li>
<li>Konqueror</li>
</ul>
  • gives us:

    Web browsers

    • Firefox
    • Chrome
    • Safari
    • IE
    • Opera
    • Konqueror

Page 7

Images

  • Use the <img> tag
  • e.g. 
    <img src="myphoto.jpg" alt="My picture!" />
  • This loads the image myphoto.jpg
  • alt specifies alternative text to display if images are turned off

Page 8

Links

  • Used to link to other pages - the key feature of the World Wide Web
  • Use the <a> (anchor) tag
  • e.g. 
    <a href="http://www.google.com"> Go to Google</a>
  • This provides a link to Google when the user clicks on Go to Google

Page 9

Forms

  • Forms are used to gather information from the user
  • Declared using the <form> tag
  • e.g. 
    <form method="get" action="">
  • Everything between <form> and </form> treated as part of the form
  • The action is the page on the website which processes the form data
  • If it's not written, use the blank string ""

Page 10

Types of form input elements

These are all used to gather data from the user in different ways:

  • Text fields
  • Radio buttons
  • Drop down lists
  • Checkboxes
  • Text areas

Page 11

Input elements

  • Specified with the <input> tag
  • Each input element has a name attribute, which uniquely identifies it

Page 12

Text field

  • Allows the user to enter text
  • Example of text field: 
    <input name="email" />
  • Here, the text field has a name of email; this name uniquely identifies it from all other text fields

Page 13

Drop down lists (select boxes)

  • These are used to make one of many choices
  • Preferred to text boxes (more user-friendly) when a user can only make one of a limited number of choices
  • Example: 
    <select name="card" >
<option value="switch"> Switch </option>
<option value="mastercard"> Mastercard </option> 
<option value="visa"> Visa </option>
</select>
  • Here, the select box has a name of card; this name uniquely identifies it from all other input elements

Page 16

Structured Form with Fieldset and Labels

  • It's generally recommended to structure your form by placing the form elements inside a fieldset with a legend explaining what the form's purpose is
  • Also it is good to explicitly link the labels to the corresponding form fields using the label tag
  • e.g. the previous example: 
    <!DOCTYPE html>
<html>
<head>
<title>My Home Page</title>
</head>
<body>

<form method="get" action="processform.php">
<fieldset>
<legend>Sign up</legend>

<label for="firstname">Enter your first name:</label>
<input name="firstname" id="firstname"/> <br/>
<label for="lastname">Enter your last name:</label>
<input name="lastname" id="lastname" /> <br/>
<label for="card">Payment Method?  </label>
<select name="card" id="card" >
<option value="switch"> Switch </option>
<option value="mastercard"> Mastercard </option> 
<option value="visa"> Visa </option>
</select>
<input type="submit" value="Go!"/>

</fieldset>
</form>

</body>
</html>
  • Note how the "for" attribute of a label references the id of the corresponding page element, not the name
  • When a user clicks on the label, the corresponding element will be focused

Page 17

Cascading Style Sheets (CSS)

  • A separate language to HTML
  • Controls look and feel of a web page or website - e.g. colours, fonts
  • (HTML controls the content and structure of a page)
  • Typically resides in a separate file to HTML pages
  • We can then import a stylesheet into a web page
  • The web page's look and feel will then be determined by the stylesheet

Page 18

A simple Cascading Style Sheet

h1 { color : blue  }
  • This Cascading Style Sheet sets all <h1> headings to blue
  • Each line of CSS defines the style of a particular tag
  • The form of each line is 
    tag { attribute : value }
  • So this CSS is setting the color attribute of <h1> headings to blue
  • The attributes (font-size, color etc) have different names to the HTML equivalents

Page 19

Setting multiple style attributes

  • We can set multiple attributes of one element
  • Each attribute:value pair is separated by a semicolon
  • e.g. 
    h1 { color : blue; font-size: 200% }
  • This will make all <h1> headings blue, and double normal font size (200%)

Page 20

Styling multiple page elements

  • We can style multiple page elements in one stylesheet
  • e.g 
    h1 { color : blue; font-size: 200% } 
h2 { background-color: green; font-size : 150% }
  • This will make all <h1> headings blue, and double normal font size (200%)
  • ... and give all <h2> headings a green background, and 150% normal font size

Page 22

How do we create a CSS file?

  • Save the CSS stylesheet in a file with extension .css e.g. simple.css
  • Add a link to the stylesheet in the web page's <head> section, e.g. 
    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html>
<head>
<link rel="stylesheet" type="text/css" href="simple.css"/>
</head>
<body>
...
  • This will set the style of the current web page to the style defined in the stylesheet simple.css
  • Thus any number of pages can easily have the same stylesheet applied to them
  • ... and all will have the same look and feel

Page 23

Multiple elements, same style

  • We can give multiple elements the same style by using a comma e.g. 
    h1,h2,h3 { color : blue }
  • This will set the colour of all <h1>, <h2> and <h3> headings to blue

Page 24

Labelling individual page elements

  • We may wish to give individual page elements a particular style
  • We can do this by giving a unique element an ID: 
    <p id="para1">First paragraph, blah blah blah...</p>

Page 25

ID example

  • HTML: 
    <p id="para1">First paragraph, blah blah blah...</p>
  • CSS: 
    #para1 { background-color: yellow }
  • We give our paragraph a unique label, "para1"
  • ... and in the CSS we give our paragraph, identified by the label "para1", a yellow background
  • Remember, in the CSS, dot for class, hash for ID!

Page 26

CSS classes

  • Used for styles independent of any particular tag
  • Used when we want to give a group of page elements a common style
  • id differs from class:
    • id defines a single page element
    • class defines a whole group of page elements which share a common style

Page 27

CSS classes example

  • Create an important class which can be applied to any tag
  • In this case, we leave out the tag in the CSS and just write the class
  • CSS: 
    .important { color : red }
  • HTML: 
    <h3 class="important">SPECIAL NOTICE!</h3>
<p class="important" >
Don't forget to hand in your assignment this week!</p>
  • In this example, both paragraph and heading will appear red

Page 28

CSS Positioning

  • With CSS, we can position our page elements at exact places on the page
  • Used for page layout

Page 29

Floating elements

  • A floating page element is one which will "float" to the left or to the right of its containing element. Any content in the containing element "flows" round it.

  • Note how the coloured element is to the left of its containing element and the text "flows" round it. This is achieved by adding a float property with a value of left to the CSS: 
    #elem1 
{
    width: 10%;
    background-color: red;
    color: white;
    float: left;
}
    Similarly, floating an element to the right can be achieved with float:right: 
    #elem1 
{
    width: 10%;
    background-color: red;
    color: white;
    float: right;
}

Page 30

Adjacent floated elements

  • A common scenario we might wish to achieve is to have several block elements, next to each other (by default, one block element would appear below the previous one). This can be achieved by applying a float: left (or float: right if we want them to be right-justified) to each element which will give this result:

  • We give the first element a float: left so it will float to the left of any subsequent content. So the second element wraps round the first element, in other words, appears to the right of it.
  • Next, this second element itself has a float: left property so a third element will wrap round it and appear to the right of it.
  • Assuming the three elements have IDs of elem1, elem2 and elem3 and they are to occupy 10% of the width of their containing element, this CSS would achieve the above effect: 
    #elem1, #elem2, #elem3
{
    float: left;
    width: 10%;
    background-color: red;
    color: white;
}

Typical sidebar / main content area layout using floats

  • Make the sidebar float to the left, giving it a given percentage width of the page. If the sidebar contains images (e.g. a logo) it will be better to give it a pixel width instead (and perhaps use CSS media queries to have an entirely different layout for mobile devices)
  • If that was all that was done, the main content area would wrap round the sidebar in an undesirable manner, e.g: as on this diagram:

  • What we want is for the whole of the main content area to appear to the right of the sidebar. This can be achieved by giving the main content area a margin-left (left margin) equal to at least the specified width of the sidebar. This margin, will shift the content of the main element over to the right, and it will appear correctly. When specifying margins, floating elements are ignored when calculating where to place the content, so the content of the main element will appear immediately adjacent to the floating element rather than the specified margin width to the right of it. Page 32

Clearing elements

  • There may be situations where we want to float an element to the left or to the right, but to have the adjacent content below the element, rather than wrapping round, as shown below:
  • In this case we place the adjacent content in its own section (elem2 here) and give this section a clear property. There are three values for clear:
    • clear:left The element will appear below any adjacent left floated element.
    • clear:right The element will appear below any adjacent right floated element.
    • clear:both The element will appear below both left and right floated elements.
    For example the layout above could be achieved with this CSS, where elem2 contains the content we wish to appear below elem1: 
    #elem1 
{
    float: left;
    width: 10%;
    background-color: red;
    color: white;
}

#elem2 
{
    clear: left;
}
